Vue + Socket.io 实现消息推送功能

posted 2020-06-19 in Vue ， 消息推送 count： 10k readingTime ≈ 9 分钟

前言

　　公司湖州企业码项目中有一个功能，第三方机构申请查看企业码的企业金融档案、企业亩均档案、企业证照档案的信息，企业码收到申请后进行审批，同意或者拒绝之后给第三方机构一个反馈。该功能的实现需要用到双向通信的消息推送，就是 WebSocket 的双向通信。
　　对于 WebSocket，可以查看WebSocket介绍，这篇文章详细介绍了 WebSocket 和 HTTP 的区别、WebSocket 实现的原理和客户端的API，所以在这里就不在赘述 WebSocket 的内容。

Scoket.io

简介

　　Socket.io 是一个 WebSocket 库，包括了客户端的 js 和服务器端的 nodejs，它的目标是构建可以在不同浏览器和移动设备上使用的实时应用。它会自动根据浏览器从 WebSocket、AJAX 长轮询、Iframe 流等等各种方式中选择最佳的方式来实现网络实时应用，非常方便和人性化，而且支持的浏览器最低达 IE5.5。

服务端

　　这里后端使用的框架是 Express，socket.io 将自身绑定到 Express 的进程中去，其中最重要的事件就是 connection 和 disconnect。它们是框架本身定义的系统事件，也就意味着它是自然就存在的不需要我们自定义，当然还有其它系统事件，但很少会用得到。

客户端

emit 和 on

  1. socket.emit(eventName[, ...args][,ack])
    触发一个事件，eventName 是触发的事件名；args 携带的参数，可以有多个参数；ack 参数是用来响应服务器用来确认消息的应答。
  2. socket.on(eventName,callback)
    监听一个事件，eventName 是监听的事件名；callback 监听事件的回调函数。

broadcast 广播

　　broadcast 默认是向所有的 socket 连接进行广播，但是不包括发送者自身，如果自己也打算接收消息的话，需要给自己单独发送。

namespace 命名空间

　　如果你想隔离作用域，或者划分业务模块，namespace 是个有效的法子。namespace 相当于建立新的频道，你可以在一个 socket.io 服务上面隔离不同的连接，事件和中间件。默认的连接也是有 namespace 的，那就是 /。

  • 使用命名空间的方式一：直接在链接后面加子域名，这种其实用的还是同一个 sokcet 服务进程，可以看成是软隔离吧。
  • 使用命名空间的方式二： path 参数，这种就是实打实的重新起了一个 socket 服务了。

rooms

　　 每一个 socket 连接都会有一个独一无二的标志，那就是 socket.id，我们就是通过 id 来区分不同连接的。除此之外，socket.id 本身也是房间 room 的标志，通俗讲，每个 socket 连接自身都拥有一间房 room。那么我们就可以给这个 room 发送消息，还有如果你加入了房间，就能接受到房间里的广播信息。当然你可以自定义 room ，让 socket 连接加入或离开。还有如果 socket 断开连接，也就是 disconnect 后，它会被自动移出room。

　　而这就是实现 单独聊天 和 群组聊天 的基础，来看一下对应的api：

  • socket.join(rooms[, callback])：加入房间
  • socket.leave(room[, callback]) ：离开房间
  • socket.to(room)： 给房间发送消息

WebSocket 和 Scoket.io 比较

　　WebSocket 是 HTML5 最新提出的规范，虽然主流浏览器都已经支持，但仍然可能有不兼容的情况，为了兼容所有浏览器，给程序员提供一致的编程体验，Socket.io 将 WebSocket、AJAX 和其它的通信方式全部封装成了统一的通信接口，也就是说，我们在使用 Socket.io 时，不用担心兼容问题，底层会自动选用最佳的通信方式。因此说，WebSocket 是 Socket.io 的一个子集。
　　这也是最终选择使用　Socket.io　实现该功能的原因。
