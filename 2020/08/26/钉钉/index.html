H5微应用

开发H5微应用

详情请查看官方文档：开发H5微应用

这里需要注意的是企业的 CorpId 获取：

  • 方法一：在开发者后台首页就可以看到
  • 方法二：在配置应用信息的时候，对应应用 > 基础信息 > 开发管理页面，在应用首页地址和 PC 端首页地址中使用 $CORPID$ 做为参数占位符，当用户在工作台打开应用时，钉钉容器会将 $CORPID$ 替换为当前访问用户的企业CorpId。详情查看官方文档：获取企业CorpId

H5微应用免登

详情请查看官方文档：H5微应用免登流程

  1. 安装钉钉 API
  2. 引入钉钉 API
  3. 获取微应用免登授权码，再根据免登授权码 code 登录应用

单聊酷应用

详情请参考官方文档：开发单聊酷应用

单聊酷应用就是在开发 H5 微应用的基础上做一些配置，比如设置消息推送、配置单聊酷应用的基本信息、安装和卸载单聊酷应用等，详情看单聊酷应用开发流程。

这里需要注意的是在配置单聊酷应用的基本信息时，填写的酷应用入口，有桌面端访问地址和移动端访问地址。
图片右上的应用点击访问的是移动端地址，图片左下的应用点击访问的是桌面端地址。

  • 移动端访问地址和配置应用信息的时候，对应应用 > 基础信息 > 开发管理页面的首页地址和 PC 端首页地址保持一致即可；
  • 桌面端访问地址比较特殊，如果配置的地址和移动端访问地址一样的话，当酷应用安装完成后，在 PC 端钉钉单聊窗口点击酷应用，会在外部浏览器打开该链接，而不是在钉钉内部打开。
    解决方法查看点击工作通知跳转到钉钉微应用，通过 dingtalk 链接的方式实现在 PC 端钉钉内部打开酷应用。如果跳转需要带参数，可以在 redirect_url 的链接地址中带上。

扫码登录第三方网站

具体官方文档查看扫码登录第三方网站，官方介绍了两种方法，

  1. 使用钉钉提供的页面登录授权
  2. 内嵌二维码方式登录授权

这里采用了方法二：支持网站将钉钉登录二维码内嵌到自己页面中。用户使用钉钉扫码登录后 JS 会将 loginTmpCode 返回给网站。JS 钉钉登录主要用途：网站希望用户在网站内就能完成登录，无需跳转到钉钉域下登录后再返回，提升钉钉登录的流畅性与成功率。

  1. 引入钉钉扫码登录JSSDK。
  2. 创建一个div，用于显示钉钉带过来的二维码
  3. login页面用户扫码-扫码成功拿到code-跳转 login 页面调接口获取用户信息然后登录去首页
