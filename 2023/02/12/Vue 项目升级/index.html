基于 vue-cli 将 webpack 3 升级到 4

  1. Node.js
    webpack4 支持的 Node.js 的最低版本是 V6，所以需要更新 Node.js 至 V6 或者更高版本。
  2. 卸载 webpack 并安装4版本。
  3. 安装 webpack-cli，webpack-cli 包从 webpack 中分离出来，所以要单独安装。
  4. 升级 vue-loader、webpack-dev-server，webpack4之后不允许混用 export 和 import，所以需要将 vue-loader 升级。
  5. 升级项目中的其他主要插件：html-webpack-plugin、eslint-loader 等。
  6. 设置 mode 模式，生产环境-production，开发环境- development，无-none。
  7. CommonsChunkPlugin 被移除，可以使用 optimization.splitChunks 来代替。
  8. 移除 extract-text-webpack-plugin（将 css 和 js 分离），用 mini-css-extract-plugin 代替。

总结：webpack3 到webpack4 的升级，主要做了以下这些事情：

  • 升级依赖：升级 webpack version为 webpack4，升级 vue-loader，升级 html-webpack-plugin、eslint-loader 等插件；
  • 代码分割：移除 CommonsChunkPlugin；引l入 SplitChunksPlugin，设置 mode 属性为 production，optimization 拆分出 webpack3中的 manifest 和 vendors；
  • 压缩css：移除 extract-text-webpack-plugin；引入 mini-css-extract-plugin，使用 mini-css-extract-plugin 的loader重新配置 sass-loader、postcss-loader和css-loader，它会为每个包含 css 的 js 文件单独构建一个 css 文件；
  • 代码热更新：升级 webpack-cli 和 webpack-dev-server， 并且设置 mode 为 development；
  • 构建加速：替换 HappyPack 为 thread-loader，多线程式本地构建和生产构建，因为 HappyPack 无法加速 vue-loader15。
