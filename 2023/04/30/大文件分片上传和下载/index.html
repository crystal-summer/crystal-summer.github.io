大文件分片上传和下载

posted 2023-04-30 in Web ， JS count： 13k readingTime ≈ 12 分钟

前端文件流操作

在前端开发中，文件流操作是指通过数据流的方式处理文件，对文件进行读取、写入和展示等操作。

数据流和文件处理的基本概念

数据流是指连续的数据序列，可以从一个源传输到另一个目的地。在前端开发中，文件可以被看作数据流的一种形式，可以通过数据流的方式进行处理。文件处理涉及读取和写入文件的操作，包括读取文件的内容、写入数据到文件，以及对文件进行删除、重命名等操作。

Blob 对象和 ArrayBuffer：处理二进制数据

在前端处理文件时，经常需要处理二进制数据。

  • Blob（Binary Large Object）对象是用来表示二进制数据的一个接口，可以存储大量的二进制数据。Blob 对象可以通过构造函数进行创建，也可以通过其他 API 生成，例如通过 FormData 对象获取上传的文件。
  • ArrayBuffer 是 JavaScript 中的一个对象类型，用于表示一个通用的、固定长度的二进制数据缓冲区。我们可以通过 ArrayBuffer 来操作和处理文件的二进制数据。

使用 FileReader 进行文件读取

FileReader 是前端浏览器提供的一个 API，用于读取文件内容。通过 FileReader，我们可以通过异步方式读取文件，并将文件内容转换为可用的数据形式，比如文本数据或二进制数据。FileReader 提供了一些读取文件的方法，例如 readAsText()、readAsArrayBuffer() 等，可以根据需要选择合适的方法来读取文件内容。

使用 Blob 实现切片文件合并

使用 Blob 对象存储所有的切片数据，创建一个临时的 URL 对象用于下载，并通过动态创建 <a> 元素模拟点击下载。

大文件切片上传

文件唯一标识

选择文件后，需要给每一个文件一个唯一标识，所以需要对文件进行哈希计算，得到的哈希值就是文件的唯一标识，这个唯一标识可以用来判断文件是否上传过或者判断文件是否已经在上传，只上传了一半。

spark-md5是一个用 JavaScript 编写的快速、高效的 md5 库，可以用它来实现创建文件的唯一标识。

实现过程

将文件分片，然后对每一个分片进行哈希计算，最后将所有分片计算出来的 md5 值整合，用作整个文件的唯一标识。

问题

将文件分片，然后对每一个分片进行哈希计算，最后将所有分片计算出来的 md5 值整合，用作整个文件的唯一标识。
但是如果文件很大的话，分片进行哈希计算，需要的时间会很久。

优化

为了解决哈希计算大文件很慢的问题，使用 web worker 来优化。

  • 使用 web worker 创建多一个子线程计算；
  • 子线程里，使用 File.slice 切片再追加分片计算哈希；
  • 线程里，只追加计算第一片和最后一片的 md5。

分片上传

初始化文件信息

循环 fileList，初始化每个文件的信息，比如 md5 值、文件名称、状态、文件大小、所有切片数据、正在上传的切片数据、请求完成的个数、报错的个数和上传进度等等。

计算切割次数

切割文件

循环切割次数，使用file.slice()方法将文件进行切割，整合切片信息，比如切片的 md5 值、切片的文件大小等等，并将所有的切片信息全部用数组allData保存到文件信息里面；

秒传

如果该文件之前已经上传过，后台已经存了一份，就没必要再次上传了，直接返回上传成功。

秒传的关键在于文件的唯一标识。

需要后端配合，后端提供一个接口，前端调用这个接口可以知道该文件有没有上传过，如果上传过了，前端直接将该文件的状态设置成上传成功状态，并将上传进度直接设置成100，以达到秒传的效果。

断点续传

断点续传顾名思义即断点 + 续传，想用户体验好就要可以断点续传：
方法一：

  • 在前端，可以使用 localForage 来存储上传的切片信息，包括上传的切片索引、切片大小等。
  • 每次上传前，先检查本地存储中是否存在上传的切片信息，若存在，则取出正在上传中、上传失败、上传中断的文件，从断点处继续上传。

localForage：是一个 JavaScript 库，通过类似 localStorage API 的异步存储来改进 Web 应用程序的离线体验。它能存储多种类型的数据，而不仅仅是字符串。它有一个优雅降级策略，若浏览器不支持 IndexedDB 或 WebSQL，则使用 localStorage，在所有主流浏览器中都可用。具体内容查看localForage 中文文档

这个方案有一个缺陷，如果换了个浏览器就失去了记忆的效果，所以一般选方法二。

方法二：

  • 在后端，可以使用一个临时文件夹或数据库来记录已接收到的切片信息，包括已上传的切片索引、切片大小等。
  • 后端写一个接口，该接口可以返回文件已上传的分片信息。
  • 前端调用该接口获取已上传的分片信息，对文件进行过滤处理，直接上传没有上传过的分片。

切片上传-slicesUpdate

  1. 浏览器同域名最大并发数限制6

  • 文件分片处理好之后，就要将所有分片发送请求给后端，如果使用Promise.all()并发请求接口，文件很大的话，切片有很多个，几百上千个切片请求瞬间浏览器卡死或者接口超时自动取消。因为浏览器同一域名同一时间是有最大请求并发数限制，解决这个问题的核心就是永远不要让并发请求数超过6。

  • 在上传切片到后端之前，先循环获取所有上传任务中正在解析文件或者正在上传的文件isTaskArrIng，通过Math.ceil(6/isTaskArrIng.length)实时动态获取并发请求数。

  1. 单个切片请求

  • 在调用后端接口上传切片的时候，使用axios cancel()方法，将该方法保存到切片信息中，用来实现暂停上传功能；

  • 如果接口调用失败，记录文件失败数taskArrItem.errNumber，如果超过三次，直接中断请求，如果没有超过三次，递归再次重新上传切片；

  • 切片上传成功之后，记录文件完成数taskArrItem.finishNumber，如果完成数和切片数相等finishNumber === sliceNumber，请求后端接口合并文件；如果不相等，递归调用分片上传方法slicesUpdate继续上传，直到所有的切片上传完成。

暂停上传（断点）

暂停上传，也就是断点，在上传切片的时候每个切片保留了axios cancel()方法，调用这个方法就可以实现中断请求。

恢复上传（续传）

恢复上传，也就是续传。续传的话需要调用后端接口查询已上传的分片信息，然后对文件的切片数据做过滤，这部分操作在分片上传的逻辑里。

更新进度条

上传文件的过程中要实时的显示文件的上传进度，这样就很直观的知道文件还差多少上传完成。

很多文章示范都喜欢用 onUploadProgress 这个方法，其实这个方法返回的进度只是文件从前端上传到服务器这一过程的进度而已，文件完全上传到了服务器 !== 上传成功，因为后端报错造成上传失败。

可以使用文件信息里的切片上传完成数和总切片数来实现进度条。

大文件切片下载

方案

实现客户端切片下载的基本方案如下：

  1. 服务器端将大文件切割成多个切片，并为每个切片生成唯一的标识符。
  2. 客户端发送请求获取切片列表，同时开始下载第一个切片。
  3. 客户端在下载过程中，根据切片列表发起并发请求下载其他切片，并逐渐拼接合并下载的数据。
  4. 当所有切片都下载完成后，客户端将下载的数据合并为完整的文件。

切片上传总结

  1. 文件唯一标识：上传文件时，需要循环文件，给每个文件创建唯一标识，使用 spark-md5 实现，将文件切片，每个切片使用 spark-md5 创建出哈希值，最后将所有切片的哈希值整合，作为文件的唯一标识，但是这样做，遇到大文件，消耗的时间长，使用 web worker来改善，并在子进程里面使用 File.slice 来进行切片，只对第一个和最后一个切片计算和整合，这样可以大大缩减时间。
  2. 文件重复上传处理：如果有的文件重复上传，即在 taskArr 中存在，需要做出判断，如果是正在上传中，删掉最后一个重复上传的任务；如果是暂停或者中断，则将最后一个重复的任务删除，并继续上传原有的任务。
  3. 断点续传：调用后端接口，如果返回该文件的切片信息，前端做过滤处理，断点续传。（也可以使用本地存储实现断点续传，如果有的文件在 uploadingArr 中存在，说明文件已经上传了一部分，将 taskArr 中原有的任务删除，添加新的至任务列表 taskArr）。
  4. 秒传：调用后端接口，查询服务器是否存在上传的文件，如果不存在的话说明没有上传过，开始分片上传；如果存在的话说明上传过了，需要设置秒传，即将文件的状态设为已完成，进度100%；如果后台报错空间不存，需要清空服务器。
  5. 分片上传：
  • 初始化文件信息；
  • 使用Math.ceil向上取整计算切割次数；
  • 循环切割次数，使用file.slice()方法将文件进行切割，整合切片信息；
  • 切片上传，需要注意浏览器同一域名同一时间是有最大请求并发数限制6个，先循环获取所有上传任务中正在解析文件或者正在上传的文件isTaskArrIng，通过Math.ceil(6/isTaskArrIng.length)实时动态获取并发请求数；
  • 单个切片请求，使用axios cancel()方法，将该方法保存到切片信息中，用来实现暂停上传功能；如果接口调用失败，记录文件失败数taskArrItem.errNumber，如果超过三次，直接中断请求，如果没有超过三次，递归再次重新上传切片；切片上传成功之后，记录文件完成数taskArrItem.finishNumber，如果完成数和切片数相等finishNumber === sliceNumber，请求后端接口合并文件；如果不相等，递归调用分片上传方法slicesUpdate继续上传，直到所有的切片上传完成。
  1. 暂停上传（断点）：在上传切片的时候每个切片保留了axios cancel()方法，调用这个方法就可以实现中断请求。
  2. 恢复上传（续传）：续传的话需要调用后端接口查询已上传的分片信息，然后对文件的切片数据做过滤，这部分操作在分片上传的逻辑里。
  3. 设置进度条：很多文章示范都喜欢用 onUploadProgress 这个方法，但是文件完全上传到了服务器 !== 上传成功，使用文件信息里的切片上传完成数和总切片数来实现进度条。
  4. 合并切片：如果完成数和切片数相等finishNumber === sliceNumber，请求后端接口合并文件。
