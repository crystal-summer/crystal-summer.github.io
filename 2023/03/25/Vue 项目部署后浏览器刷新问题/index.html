Vue 项目部署后浏览器刷新问题

posted 2023-03-25 in Vue ， 打包 count： 4.3k readingTime ≈ 4 分钟

问题

Vue 项目打包后，会给静态文件后面加上 hash 值，每次打包后 hash 值都是不一样的，由于缓存原因，需要手动刷新浏览器，静态文件才会是最新的打包的文件。但是在实际使用过程中，用户往往不知道发布了新的版本，页面停留在上一个版本的状态，此时操作页面可能会出现问题，所以最好的解决方法就是提示用户刷新浏览器，或者自动刷新。

解决方案

  1. 前后端配合定义一个版本接口
    这种场景和 APP 版本更新相似，每次上线都定义一个版本号，这个版本号通过后端接口进行修改和获取，这个方案需要前后端配合。
  2. WebSocket
    需要一个 WebSocket 服务, 还需要后端配合。
  3. 对比 html 文件里引入的 js 文件的 hash 值
    • 不支持监听静态资源的更新；
    • webpack 打包时给文件路径加 hash 的功能是可以被关闭的。
  4. 检查 Response head 的 Etag、Last-Modified 值
    • 需要服务端开启缓存；
    • 一些代理服务可能在转发 HTTP 请求和响应时在 Etag 和 Last-Modified 之前去除这些头信息，以减少网络负载和提高性能；
    • 不支持监听静态资源的更新。
  5. service worker (PWA)
    • 接入 service worker 需要成本，本地运行一个 worker 也会占用内存和 cpu 资源；
    • 它具有拦截并处理网络请求的能力，如果不熟悉，很有可能造成强缓存，手动强制刷新页面都无法更新。参考谨慎处理 Service Worker 的更新。
    • 如果非要使用的话， 建议使用 vite-plugin-pwa，插件已经内置了更新提示功能，开发也不用关心如何更新 sw.js。
  6. 使用 plugin-web-update-notification插件，是一个用来监听网页更新，并通知用户刷新页面的插件。

plugin-web-update-notification

plugin-web-update-notification，监听网页更新，并通知用户刷新页面的插件，支持 Vite、Webpack、umi。

实现原理：
以 git commit hash (也支持 package.json version、build timestamp、custom) 为版本号，打包时将版本号写入一个 json 文件，同时注入客户端运行的代码。客户端轮询服务器上的版本号（浏览器窗口的visibilitychange、focus 事件辅助），和本地作比较，如果不相同则通知用户刷新页面。

优点：

  1. 接入简单，安装插件，修改配置文件即可，不用修改业务代码（如果不自定义行为）。
  2. 支持多种版本号类型：
    • git commit hash（默认）。
    • package.json 中的 version 字段。
    • build timestamp，运行打包命令时的时间戳。
    • custom，用户自定义版本号。
  3. 预置了一个简约的更新 Notification，可以自定义文案、样式、位置。当然也可以取消默认的 Notification，监听到更新事件后自定义行为。
  4. 预置的 Notification 文案支持国际化。
  5. tiny, 注入的 js 和 css 文件压缩后不到 2kb。
  6. 完善的 ts 类型提示。

新旧版本号比对

将新旧版本号比对的方法提取为公共方法，多个地方会用到。

比对时机

常见的比对时机有：

  • 在路由拦截beforeEach钩子里比对；
  • 通过定时器轮询比对。

这里选择在路由拦截beforeEach钩子里判断版本号是否一致，不一致通知用户刷新浏览器或者自动刷新浏览器。

这里之所以是在 beforeEach 钩子里处理是因为 afterEach 和 beforeResolve 是在 beforeEach 执行之后执行，当服务器文件已经更新了，然后本地还没更新的情况下，是加载不到即将要跳转的页面文件的。

优化

问题：当用户在登录页时，服务端更新了版本，所以用户在输入用户名和密码之后，点击登录，显示登录成功，正常是应该直接跳到首页，但是此时跳转前检测到版本更新，所以会原地刷新一下获取最新资源，导致页面又恢复到了登录之前的状态，用户需要重新登录。
解决方法：在 App.vue 页面，用户打开/聚焦页面的时候，先更新一下版本号。

总结

针对 Vue 项目打包部署后必须刷新浏览器才能拿到最新资源的问题，大多数的解决方案都是通过定义版本号来解决，不同的地方在于版本号如何生成，比较新旧版本号的时机（路由的 beforeEach，全局监听 click，全局监听 visibilitychange、focus 等）等。
