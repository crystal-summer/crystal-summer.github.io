Git 与 sourceTree 常用操作

posted 2019-09-07 in Web ， Git count： 2k readingTime ≈ 2 分钟

新建分支

比如：以master分支为基础新建一个dev分支

  1. Git
  2. sourceTree
    将当前分支切换到master，点击顶部操作栏的分支，选择新分支，输入新分支名称dev，点击确定就创建了本地分支，并且已经检出切换到dev分支，然后点击顶部操作栏的推送，将本地分支推送到远程。

删除分支

  1. Git
  2. sourceTree
    直接在本地分支和远程分支上面右键单击选择删除即可。

分支合并

比如：将dev分支的代码合并到master分支

  1. Git
  2. sourceTree
    先切换到master分支，然后右键单击dev分支，选择合并dev至当前分支，然后再点击顶部操作栏的推送即可。

拉取远程分支

比如：gitlab上新建了一个分支test，但是在本地查询不到，无法切换分支、

  1. Git
  2. sourceTree
    右键单击远程仓库(origin)，选择从origin获取。

reset 和 reverse

  1. reset：回退到指定的版本，最新版到指定版本之间提交的修改都被删除了；
  2. reverse：还原某次的提交，只会删除当前之地的版本的内容，并且产生新的版本号。

回滚

比如：本地多次提交代码，多次推送到远程，现在想回到某一次的提交

  1. Git
  2. sourceTree
    （1）在History中找到想要回到的某次提交，右键单击选择重置当前分支到此次提交，选择使用模式-强行合并；
    （2）在History中找到最新的提交，右键单击选择重置当前分支到此次提交，选择使用模式-软合并；
    （3）此时在暂存区中即可获取到从历史节点到当前节点的所有修改记录，这个时候可以根据需求进行修改再提交。
  • 软合并：是指将此次提交回滚到指定提交位置，但这个过程中会将修改过的文件暂存到暂存区；
  • 混合合并：是指将此次提交回滚到指定的位置，但这个过程中不会将修改过的文件暂存到暂存区，而是将修改过的文件存放在未暂存文件区；
  • 强行合并：是指将此次提交回滚到指定的位置，但这个过程中将直接丢弃之前修改的所有文件，因此在选取此种合并时需要考虑清楚，避免一些不必要的麻烦。

将另一个分支的某次提交合并到当前分支

  1. Git
  2. sourceTree
    在History中找到提交的记录，右键单击，选择遴选（弹框有两个多选框选中，表示合并成功直接提交）即可。
