盒子模型

  • W3C 标准盒子模型
    box width = content width
    box height = content height
  • IE 盒子模型
    box width = content width + padding + border
    box height = content height+ padding + border

在编写页面代码时应尽量使用标准的 W3C 模型(需在页面中声明 DOCTYPE 类型)，这样可以避免多个浏览器对同一页面的不兼容。

因为若不声明 DOCTYPE 类型，IE 浏览器会将盒子模型解释为 IE 盒子模型，FireFox 等会将其解释为 W3C 盒子模型；若在页面中声明了 DOCTYPE 类型，所有的浏览器都会把盒模型解释为 W3C 盒模型。

盒子模型的相互转化
box-sizing:content-box：按照 W3C 标准盒子模型显示；
box-sizing:border-box：按照 IE 盒子模型显示，通过已设定的宽度和高度分别减去边框和内边距才能得到内容的宽度和高度。

margin 重叠

margin 的重叠只产生在普通流文档的 上下 margin 之间，左右 margin 不会发生重叠，重叠的计算方法如下：

  • 全部都为正值，取最大者；
  • 不全是正值，则都取绝对值，然后用正值减去最大值；
  • 没有正值，则都取绝对值，然后用0减去最大值。

发生重叠的情况：

  • 相邻兄弟节点 div1 的 margin-bottom:30px 和 div2 的 margin-top:20px 发生重叠，本该 50px 的 margin 重叠为 30px。
  • 父子节点，没有内容分隔 父节点和子节点的左右 margin 不重叠，上下 margin 重叠为 30px。
  • 空的块状元素 此时上下 margin 会重叠，最终形成一个高度为 30px 的空间。

解决方法：

  1. 内节点设置 padding；
  2. 内节点设置透明边框 border:1px solid transparent;；
  3. 内节点设置绝对定位 position:absolute; 脱离普通文档流；
  4. 内节点设置 float 脱离普通文档流；
  5. 内节点设置 display:inline-block; 脱离普通文档流；
  6. 外节点设置 overflow:hidden;。

居中样式

  1. display:flex + margin:auto
  2. 定位 + margin:auto
  3. 定位 + translate
  4. flex 弹性布局
  5. grid 网格布局
  6. 父节点 table-cell 垂直居中，子节点 margin:auto 水平居中
